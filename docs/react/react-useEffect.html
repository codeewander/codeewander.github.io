[react] useEffect

Key points#

  • 透過 useEffect 可以在 function component 中執行帶有副作用（side effects）的方法，像是：向伺服器請求資料（data fetching）、訂閱（subscription）、手動操作 DOM
  • 執行時間點為 React 更新完 DOM 之後。若有回傳 cleanup function 的話，則會先執行，接著才執行該次的 effect
  • 每一次元件重新render時 useEffect 內的函式都是新的。
  • useEffect 的使用時機類似在傳統 class component 中使用 componentDidMount, componentDidUpdate 和 componentWillUnmount 的生命週期
  • 要注意第一次執行的時間點，如果第一次不想要執行要 return 掉
  • useEffect 的回傳值只能是空或者是清理函式（cleanup function）
  • 要注意 dependencies 陣列中的變數，它們的改變會觸發 useEffect 執行
  • dependencies array 是透過 Object.is() 來比較兩個值是否相等。要注意的是如果是不同 reference 的物件則不相等

基本用法#

兩種常見的副作用，一種需要被清理（cleanup），一種則不用

不需要被清理的作用（Effects Without Cleanup）#

做完就不用再去管它的副作用，像是：API請求、手動操作 DOM等

function Example() {  const [count, setCount] = useState(0);
  useEffect(() => {    document.title = `You clicked ${count} times`;  },[count])
  return (    <div>      <p>You clicked {count} times</p>      <button onClick={() => setCount(count + 1)}>Click me</button>    </div>  );}

需要被清理的作用（Effects with Cleanup)#

像是訂閱（subscription）一些外部資料，就需要做清理，如此才可以避免記憶體洩漏（memory leak）的問題。 在 useEffect 中，可以回傳一個清理函式，它會在「每次元件render之後」，以及「元件 unmount」時被執行。

import React, { useState, useEffect } from 'react';
function Counter() {  const [count, setCount] = useState(0);  console.log('Invoke Counter');
  useEffect(() => {    document.title = `You clicked ${count} times`;    console.log('useEffect', count);
    return () => {      console.log('useEffect cleanup', count);    };  });
  return (    <div>      {console.log('render')}      <p>You click {count} times</p>      <button onClick={() => setCount(count + 1)}>Add</button>    </div>  );}
export default Counter;

以下是 console 執行的時間：

# 第一次元件渲染Invoke CounterrenderuseEffect 0
# 元件更新Invoke CounterrenderuseEffect cleanup 0   // 裡面是上一次 useEffect 取得的 state 和 propsuseEffect 1           // 這次 useEffect 取得的 state 和 props
＃ 元件更新Invoke CounterrenderuseEffect cleanup 1  // 裡面是上一次 useEffect 取得的 state 和 propsuseEffect 2  // 這次 useEffect 取得的 state 和 props
# 元件 unmount 時useEffect cleanup 2

Q: 為什麼有時候 react 元件會渲染兩次？#

可能是沒注意到使用了React StrictMode的關係(官方回覆)。在 concurrent mode 和 strict mode 的開發階段，React故意渲染兩次，幫助開發者debugger，因此，沒有寫在useEffect 的中的異步副作用都會被觸發兩次。但是第二次渲染中，如果是 console 則不會渲染兩次，因為React調用了disableLogs(stack overflow回覆)。
